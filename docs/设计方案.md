## 1. 项目概述

### 1.1 定位

终端 TUI 工具，用于浏览、搜索、批量管理 OpenCode 会话历史。参考 [opencode-history](https://github.com/felix-lj-ct/opencode-history) (oh) Web Dashboard 的设计理念，在 TUI 终端中实现会话管理，并**额外提供 oh 不具备的删除、批量删除、会话类型过滤、备份、数据库收缩等管理能力**。

### 1.2 目标用户

- 使用 OpenCode AI Agent 的开发者
- 需要定期清理大量会话数据的用户
- 需要在无 VS Code / Node 环境的 Windows 机器上管理会话的用户

### 1.3 设计原则

| 原则 | 说明 |
|---|---|
| **本地优先** | 只读打开数据库（删除操作除外），不解锁文件 |
| **终端原生** | 纯 TUI 操作，无 Web 依赖，单 exe 无运行时 |
| **安全第一** | 删除前自动备份，二次确认，删除后可选择 VACUUM |
| **兼容降级** | type 列不存在时自动降级，不阻塞旧版数据库 |
| **快捷键驱动** | 全键盘操作，无需鼠标，批量操作一步到位 |

## 4. 数据模型

### 4.1 Session（会话）

| 字段 | 类型 | 说明 |
|---|---|---|
| Id | string | 会话 ID（主键） |
| Title | string | 会话标题 |
| ProjectPath | string | 关联项目路径 |
| **Type** | **string** | **会话类型（sisyphus / explore / build 等）** |
| LastUpdatedAt | DateTime | 最后更新时间 |
| MessageCount | int | 消息条数 |
| SizeBytes | long | 占用大小（字节） |
| FormattedSize | string | 只读，友好格式（KB/MB） |
| FormattedTime | string | 只读，格式 yyyy-MM-dd HH:mm |
| **DateGroup** | **string** | **只读，日期分组标签** |
| **IsMainSession** | **bool** | **只读，是否为 sisyphus 主对话** |

### 4.2 Database Schema（OpenCode 原始表）

**conversations 表：**

| 列 | 类型 | 说明 |
|---|---|---|
| `id` | TEXT PK | 会话 ID |
| `title` | TEXT | 标题 |
| `project_path` / `directory` | TEXT | 项目路径 |
| **`type` / `agent`** | **TEXT** | **会话类型** |
| `parent_id` | TEXT? | 非空 = 子 agent 会话 |
| `updated_at` | DATETIME | 最后更新时间 |
| `created_at` | DATETIME | 创建时间 |

**messages 表：**

| 列 | 说明 |
|---|---|
| `conversation_id` | 关联到 conversations.id |
| `role` | user / assistant |
| `content` | 消息内容 |
| `created_at` | 时间戳 |

抽象类型 `DateGroup` 在运行时计算，不存储在数据库中。

### 5.1 参数列表

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `--db <path>` | string | 自动探测 | 数据库路径 |
| `--backup-only` | bool | false | 仅备份退出 |
| `--purge-before <date>` | string? | null | 删除指定日期前会话 |
| `--vacuum` | bool | false | 仅 VACUUM |
| `--no-auto-backup` | bool | false | 关闭删除前自动备份 |
| `--version` | 内置 | — | 显示版本号 |
| `--help` | 内置 | — | 显示帮助 |

### 5.2 运行模式路由

```
Main(args)
  │
  ├── args 为空 → 自动定位数据库 → 启动横幅 → TUI
  │
  └── args 非空 → RootCommand.InvokeAsync(args)
       ├── --version      → 版本号
       ├── --help         → 帮助
       ├── --backup-only  → 备份 → 退出
       ├── --vacuum       → VACUUM → 退出（显示压缩对比）
       ├── --purge-before → 自动备份 → 删除 → 退出
       └── 无匹配        → TuiService.Run()
```

## 6. 交互式 TUI 设计

### 6.1 主循环架构

```
Run()
  ├── Console.CursorVisible = false
  ├── ValidateDatabase()
  ├── _dbService.Open()
  ├── Status 转圈:
  │     ├── _projectPaths = GetProjectPaths()
  │     ├── _availableTypes = GetSessionTypes()
  │     └── LoadData()
  ├── while (_running)
  │     ├── Render()       # 日期聚类 + 双行布局
  │     └── HandleInput()  # 键盘事件
  └── Console.CursorVisible = true
```

### 6.3 键盘映射

| 分类 | 按键 | 功能 |
|---|---|---|
| 导航 | `↑/↓` | 移动光标 |
| | `←/→` / `PgUp/PgDn` | 翻页 |
| | `Home/End` | 首页 / 末页 |
| 选择 | `Space` | 切换当前行选中 |
| | `A` | 全选 / 取消全选 |
| | `Shift+↑/↓` | 区间选中 |
| | `Esc` | 清除选中 |
| 操作 | `Enter` | 预览会话 |
| | `D` | 删除选中 |
| 过滤 | `S` | 搜索 |
| | `F` | 筛选（时间+项目+类型） |

| | `V` | VACUUM |
| 系统 | `F1` | 帮助 |
| | `Q` / `Ctrl+Esc` | 退出 |

### 6.4 类型过滤机制

默认 `_sessionType = "sisyphus"`，`T` 键在 null ↔ "sisyphus" 间切换。切换后清除选中并重载数据。

### 6.5 删除流程

```
按 D
  ├── 无选中 → 提示 → 返回
  └── 有选中
      ├── 展示待删预览表
      ├── AutoBackup? → CreateBackup()
      ├── 二次确认 Y/N
      │     ├── N → 取消
      │     └── Y → DeleteSessions() → 询问 VACUUM?
      │           ├── Y → ExecuteVacuum()
      │           └── N → 完成
      └── 清理选中，刷新列表
```

## 7. 数据库服务

### 7.1 DatabaseService API

```
class DatabaseService : IDisposable
{
    +string DbPath / +long DbFileSize / +bool IsConnected
    +static string GetDefaultDbPath()
    +DatabaseService(string? customPath)
    +(bool, string) ValidateDatabase()          // 文件存在 + PRAGMA integrity_check
    +bool Open() / +void Dispose()
    +int GetTotalSessionCount()
    +List<Session> GetSessions(offset, limit, keyword?, beforeDate?, project?, sessionType?)
    +int GetFilteredCount(keyword?, beforeDate?, project?, sessionType?)
    +List<string> GetProjectPaths()
    +List<string> GetSessionTypes()             // SELECT DISTINCT type
    +(bool, string) GetSessionPreview(id)
    +int DeleteSessions(List<string> ids)        // 事务
    +int PurgeSessionsBefore(DateTime cutoff)
    +(bool, long, long, string) Vacuum()        // 返回压缩前后大小
}
```

### 7.2 降级策略

`GetSessions()` 先尝试带 `type` 列的查询，如果 `SqliteException`（列不存在），自动降级到 `GetSessionsLegacy()`。

### 7.3 默认路径探测

```
GetDefaultDbPath()
  ├── USERPROFILE → ~/.local/share/opencode/opencode.db
  ├── HOME → ~/.local/share/opencode/opencode.db
  └── fallback → C:\Users\Default\.local\share\opencode\opencode.db
```

## 8. 备份管理

### 8.1 BackupService API

```
class BackupService
{
    +string BackupDirectory
    +BackupService(dbPath, customBackupDir?)
    +string CreateBackup()                         // 文件复制
    +List<BackupInfo> ListBackups()                // 按时间倒序
    +(bool, string) RestoreBackup(backupFilePath)  // 恢复前先备份当前库
    +int CleanupOldBackups(keepDays)               // 删除过期备份
}
```

### 8.2 备份命名

```
opencode_20260708_143000.db
  └──────┴────────────┴─
   库名    yyyyMMdd_HHmmss  .db
```

### 8.3 安全策略

恢复备份时先自动备份当前库，避免覆盖导致数据丢失。
