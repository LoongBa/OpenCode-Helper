## 1. 项目概述

### 1.1 定位

终端 TUI 工具，用于浏览、搜索、批量管理 OpenCode 会话历史。参考 [opencode-history](https://github.com/felix-lj-ct/opencode-history) (oh) Web Dashboard 的设计理念，在 TUI 终端中实现会话管理，并**额外提供 oh 不具备的删除、批量删除、会话类型过滤、备份、数据库收缩等管理能力**。

### 1.2 核心差异化

| 对比项    | opencode-history (oh) | 本工具                       |
| ------ | --------------------- | ------------------------- |
| 形态     | Node.js Web 服务        | 单文件 AOT exe               |
| 依赖     | Node.js >= 16         | 无（独立运行）                   |
| 核心功能   | 查看 + 一键恢复             | 查看 + 批量管理                 |
| 删除能力   | ❌ 只读                  | ✅ 删除 / 批量删除               |
| 备份管理   | ❌                     | ✅ 创建 / 查看 / 恢复 / 清理       |
| 数据库收缩  | ❌                     | ✅ VACUUM                  |
| 会话类型过滤 | ❌                     | ✅ sisyphus / explore / 全部 |
| 启动速度   | 秒级（Node 启动）           | 毫秒级（AOT 原生）               |

### 1.3 技术栈

| 项目  | 选型                                       |
| --- | ---------------------------------------- |
| 语言  | C# (.NET 10)                             |
| 发布  | Native AOT，单文件独立 exe，无需运行时               |
| CLI | System.CommandLine 2.0.0-beta4.22272.1   |
| TUI | Spectre.Console 0.50.*                   |
| 数据库 | Microsoft.Data.Sqlite 10.0.0-preview.3.* |
| 平台  | Windows x64                              |

### 1.4 数据源

OpenCode 全局 SQLite 数据库，自动定位路径：

| 平台      | 路径                                                   |
| ------- | ---------------------------------------------------- |
| Windows | `%USERPROFILE%\.local\share\opencode\opencode.db`    |
| Linux   | `~/.local/share/opencode/opencode.db`                |
| macOS   | `~/Library/Application Support/opencode/opencode.db` |

### 1.5 数据库 Schema（已知字段）

`conversations` 表：

| 列                            | 类型       | 说明                                           |
| ---------------------------- | -------- | -------------------------------------------- |
| `id`                         | TEXT PK  | 会话 ID（如 `ses_xxx`）                           |
| `title`                      | TEXT     | 会话标题                                         |
| `project_path` / `directory` | TEXT     | 所属项目路径                                       |
| `type` / `agent`             | TEXT     | 会话类型（sisyphus / explore / build / general 等） |
| `parent_id`                  | TEXT?    | 非空 = 子 agent 会话（主列表应过滤）                      |
| `updated_at`                 | DATETIME | 最后更新时间                                       |
| `created_at`                 | DATETIME | 创建时间                                         |

`messages` 表：

| 列                 | 说明                   |
| ----------------- | -------------------- |
| `conversation_id` | 关联到 conversations.id |
| `role`            | user / assistant     |
| `content`         | 消息内容（Markdown）       |
| `created_at`      | 时间戳                  |

## 3. 功能需求

### 3.1 启动与数据库

| ID   | 需求            | 说明                               |
| ---- | ------------- | -------------------------------- |
| F001 | 自动定位数据库       | 按多平台路径规则自动查找 opencode.db         |
| F002 | 自定义数据库路径      | `--db <path>` 参数覆盖默认路径           |
| F003 | 数据库完整性校验      | `PRAGMA integrity_check`，失败则禁止操作 |
| F004 | 版本信息显示        | 启动横幅显示名称 + 版本 + "使用 --help 查看"   |
| F005 | 数据库 Schema 降级 | 若 `type` 列不存在，自动降级为无类型过滤查询       |

### 3.2 会话列表展示

| ID   | 需求     | 说明                                      |
| ---- | ------ | --------------------------------------- |
| F101 | 分页加载   | 每页 20 条，`←/→` 翻页，`Home/End` 首尾页         |
| F102 | 双行布局   | 第1行：复选框 + 类型标记 + 标题；第2行：项目路径 + 时间 + 消息数 |
| F103 | 类型标记   | `●` 主对话（sisyphus）、`○` 其他（explore 等）     |
| F104 | 日期聚类分组 | 今天 → 昨天 → 本周 → 本月 → 更早，分组标题显示条数         |
| F105 | 选中状态   | 黄色 `[✓]` 表示已选中，灰色底色表示光标所在行              |
| F106 | 状态栏    | 顶部蓝条显示选中数 / 总数 / 当前类型 / 排序方向            |
| F107 | 筛选栏    | 青色显示当前搜索关键词、时间范围、项目筛选                   |
| F108 | 底部操作栏  | 灰色显示当前有效快捷键列表                           |

### 3.3 过滤与筛选

| ID       | 需求         | 快捷键           | 说明                                                     |
| -------- | ---------- | ------------- | ------------------------------------------------------ |
| F201     | 关键词搜索      | `S`           | 模糊匹配标题+项目路径，`AnsiConsole.Ask` 输入                       |
| F202     | 时间范围筛选     | `F`           | 全部 / 近1个月 / 近6个月                                       |
| F203     | 项目路径筛选     | `F`           | 列出所有不重复项目路径，选择过滤                                       |
| **F204** | **会话类型过滤** | **`T` / `F`** | **默认仅显示 sisyphus 主对话；`T` 键切换全部↔仅主对话；`F` 筛选对话框可选择具体类型** |
| F205     | 排序切换       | `R`           | 时间降序（默认）↔ 升序切换，状态栏显示 `↓`/`↑`                           |

### 3.4 会话操作

| ID       | 需求       | 快捷键                 | 说明                                              |
| -------- | -------- | ------------------- | ----------------------------------------------- |
| F301     | 会话预览     | `Enter`             | `Panel` 显示完整对话摘要（标题+项目+时间+前20条消息）               |

## 4. 命令行接口

程序支持两种运行模式：**交互式 TUI** / **纯命令行静默执行**

### 4.1 参数列表

| 参数                      | 类型      | 默认值   | 说明                      |
| ----------------------- | ------- | ----- | ----------------------- |
| `--db <path>`           | string  | 自动探测  | 自定义数据库路径                |
| `--backup-only`         | bool    | false | 仅执行全库备份后退出              |
| `--purge-before <date>` | string? | null  | 删除指定日期前所有会话（yyyy-MM-dd） |
| `--vacuum`              | bool    | false | 仅执行数据库收缩                |
| `--no-auto-backup`      | bool    | false | 关闭删除前自动备份               |
| `--version`             | 内置      | —     | 显示版本号                   |
| `--help`                | 内置      | —     | 显示帮助信息                  |

### 4.2 运行模式路由

```
Main(args)
  │
  ├── args 为空 ──→ 显示启动横幅 → 进入 TUI
  │
  └── args 非空 ──→ System.CommandLine 解析
       ├── --version    → 显示版本号
       ├── --help       → 显示帮助
       ├── --backup-only → 备份 → 退出
       ├── --vacuum     → VACUUM → 退出（显示压缩对比）
       ├── --purge-before → 可选备份 → 批量删除 → 退出
       └── 其他组合     → 进入 TUI
```

## 5. 非功能性需求

### 5.1 AOT 编译

| 需求     | 说明                                         |
| ------ | ------------------------------------------ |
| 单文件发布  | Native AOT，输出 `OpenCodeSessionManager.exe` |
| 无运行时依赖 | 无需安装 .NET Runtime                          |
| 启动速度   | 毫秒级                                        |
| 文件体积   | 约 5-8 MB                                   |
| 优化方向   | 体积优先（`OptimizationPreference=Size`）        |

### 5.2 异常处理

| 场景        | 行为                                  |
| --------- | ----------------------------------- |
| 数据库文件不存在  | 提示错误路径，退出                           |
| 数据库损坏     | `PRAGMA integrity_check` 失败，提示并禁止操作 |
| 数据库锁定     | `SqliteException` 捕获，友好提示           |
| 磁盘空间不足    | 备份/VACUUM 失败时提示                     |
| type 列不存在 | 自动降级为无类型过滤查询                        |
| 配置加载失败    | 使用默认配置，静默忽略                         |

### 5.3 安全

| 需求    | 说明                            |
| ----- | ----------------------------- |
| 只读浏览  | 浏览/预览不写数据库                    |
| 删除安全  | 删除前自动备份 + 二次确认                |
| 恢复安全  | 恢复备份前先备份当前库                   |
| 备份不覆盖 | `File.Copy(overwrite: false)` |

## 6. 用户操作流程

### 6.1 清理旧会话（典型场景）

```
1. 启动 exe
   → 显示名称+版本+提示
   → 自动定位 opencode.db
   → 加载数据（显示转圈+读秒）
   → 默认仅显示 sisyphus 主对话
   → 日期聚类展示会话列表

2. 按 S 搜索关键词找到目标会话
   或按 F 筛选时间范围 + 项目目录
   或按 T 切换显示全部类型（含 explore 等）

3. 空格 / A / Shift+↑↓ 批量选中

4. 按 D 删除
   → 展示待删预览表
   → 自动备份（如开启）
   → Y 确认删除
   → 可选 VACUUM 回收空间

5. 按 B 进入备份管理（如需存档）
   → 创建备份 / 查看历史 / 清理旧备份

6. 按 Q 退出
```

### 6.2 命令行批量清理（脚本场景）

```bash
# 清理 30 天前的所有会话（含自动备份）
OpenCodeSessionManager.exe --purge-before 2026-06-08

# 仅做备份
OpenCodeSessionManager.exe --backup-only

# 收缩数据库
OpenCodeSessionManager.exe --vacuum
```

## 7. 不做范围

以下功能在后续版本中考虑，当前不做：

| 功能                         | 原因                          |
| -------------------------- | --------------------------- |
| 项目级总览卡片（Token/费用统计）        | TUI 空间有限，且需解析消息 Token 计数    |
| 一键恢复终端（`opencode -s <id>`） | 需配置终端命令，跨平台兼容复杂             |
| 可自定义显示列                    | TUI 空间固定，灵活列布局成本高           |
| i18n（中英切换）                 | MVP 阶段仅中文，后续可加              |
| macOS/Linux 支持             | 当前面向 Windows 用户，AOT 跨平台可后续加 |
